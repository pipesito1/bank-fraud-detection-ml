# 🛡️ Bank Fraud Detection ML

<img src="assets/headline_bank.png" alt="Bank Fraud Detection Project Banner" width="750">

Machine Learning project focused on detecting fraudulent banking transactions using data analysis and classification models.

</div>

---

# 📖 Overview

Financial fraud is a major challenge for banks and financial institutions, causing billions of dollars in losses every year. Detecting suspicious transactions quickly is critical to prevent financial damage and protect customers.

This project focuses on building a **Machine Learning model capable of identifying fraudulent banking transactions** through data analysis, feature engineering, and classification algorithms.

The project also includes a **simple web interface that allows users to test transactions and see whether the model predicts fraud or not**, demonstrating how machine learning models can be integrated into real-world applications.

---

# 🎯 Objectives

- Detect fraudulent transactions using machine learning models
- Perform **data cleaning and preprocessing (ETL)**
- Conduct **Exploratory Data Analysis (EDA)**
- Train and evaluate classification models
- Identify patterns associated with fraudulent transactions
- Demonstrate the model through a **simple web application**

---

# 📊 Dataset

The dataset contains transaction-related features commonly used in fraud detection problems.

Some example variables include:

- Transaction amount  
- Transaction time  
- Account information  
- Transaction type  
- Fraud label (target variable)  

The target variable identifies whether the transaction is:

- **0 → Legitimate transaction**
- **1 → Fraudulent transaction**

Fraud detection datasets are typically **highly imbalanced**, meaning fraudulent transactions represent only a small percentage of the total transactions.

# 🔎 Project Workflow

## 1️⃣ Data Cleaning (ETL)

- Handling missing values  
- Removing duplicates  
- Formatting variables  
- Preparing the dataset for analysis  

---

## 2️⃣ Exploratory Data Analysis (EDA)

The exploratory analysis includes:

- Transaction distribution analysis  
- Fraud vs legitimate transaction comparison  
- Correlation analysis  
- Visualization of patterns in fraudulent activity  

---

## 3️⃣ Feature Engineering

Feature engineering was applied to improve model performance:

- Scaling numerical features  
- Preparing data for machine learning models  

---

## 4️⃣ Model Training

Several classification models were tested to detect fraudulent transactions:

- Logistic Regression  
- Decision Tree  
- **Random Forest (Final Selected Model)**

The **Random Forest algorithm** was selected as the final model because it provided strong performance and robustness when detecting fraudulent transactions in the dataset.

Random Forest is particularly effective in fraud detection because it:

- Handles complex feature interactions  
- Works well with imbalanced datasets  
- Reduces overfitting through ensemble learning

---

# 📈 Model Evaluation

Models were evaluated using multiple metrics:

- Accuracy  
- Precision  
- Recall  
- F1 Score  
- Confusion Matrix  

Special attention was given to **Recall**, since correctly identifying fraudulent transactions is critical in fraud detection systems.

---

# 🌐 Fraud Detection Web Demo

This project also includes a **simple web application that allows users to input transaction data and obtain a fraud prediction from the trained machine learning model.**

The web interface demonstrates how the fraud detection model can be used in a **real-world scenario**, where financial institutions need to analyze transactions quickly.

Features of the web demo:

- Input transaction information  
- Send data to the trained model  
- Receive prediction result  
- Display whether the transaction is **fraudulent or legitimate**
